## ğŸ“˜ Project Overview

This project demonstrates **autonomous path navigation** using an **A* pathfinding algorithm** running on a **Raspberry Pi (or PC)** that communicates with an **Arduino-based robot**.  
The system takes an occupancy grid map (previously generated by an ultrasonic mapping system), computes the **shortest path** between a start and goal position, and sends **movement and rotation commands** to the Arduino for real-world motion.

This project builds upon the previous **Ultrasonic Mapping System**, where the environment was scanned and saved as `lab8_map.csv`.

ğŸ§  Code Description
joshua.py
Loads map from lab8_map.csv.

Uses A* algorithm to find the optimal path:

Each node represents a cell in the occupancy grid.

Uses diagonal and straight movements with appropriate cost.

Sends movement commands to Arduino via serial (/dev/ttyACM0).

Visualizes the path over the map using Matplotlib.

Key Features

Heuristic: Euclidean distance

Obstacle threshold: 0.8

Adjustable grid size and movement delay

Automatic path visualization

sketch_nov03b.ino
Receives commands from Python through Serial communication.

Parses incoming commands (ROTATE or MOVE).

Controls DC motors using an L298N driver for:

Differential rotation (turn left/right).

Forward movement based on time or encoder feedback.

Designed for easy integration with path-planning scripts.
